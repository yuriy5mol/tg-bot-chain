# 🤖 TG Bot Chain — Генератор Telegram-ботов

Автоматическая генерация Telegram-ботов из текстового описания с помощью LangChain и OpenAI.

## Как это работает

Скрипт использует пайплайн из **4 цепочек LangChain** + **AST-валидацию**:

```
Описание бота
     │
     ▼
┌─────────────────────┐
│ 1. analysis_chain   │  Анализ задания → команды, обработчики, зависимости
│    temperature=0.1  │
└──────────┬──────────┘
           ▼
┌─────────────────────┐
│ 2. code_chain       │  Генерация полного кода бота на aiogram 3.7+
│    temperature=0.2  │
└──────────┬──────────┘
           ▼
┌─────────────────────┐
│ 3. review_chain     │  Ревью: проверка логики, структуры, API
│    temperature=0.0  │
└──────────┬──────────┘
           ▼
     validate_code()     AST-валидация: синтаксис + проверка импортов
           │
       ✅ OK? ──→ Сохраняем generated_bot.py
           │
       ❌ Нет
           ▼
┌─────────────────────┐
│ 4. fix_chain        │  Исправление конкретной ошибки (до 3 попыток)
│    temperature=0.0  │  Получает точное описание ошибки от AST
└──────────┬──────────┘
           ▼
     validate_code()  ──→ ✅ / 🔄 повтор fix_chain
```

### Защита от ошибок

- **Retry с контекстом:** при ошибке валидации код отправляется в `fix_chain` с точным описанием проблемы (`"Отсутствуют импорты: os, asyncio"`)
- **Откат:** если `review_chain` сломал рабочий код — автоматический откат к оригиналу
- **Извлечение кода:** `extract_code()` корректно извлекает Python-код, даже если LLM обернул его в markdown или добавил пояснения

## Требования

- Python 3.10+
- API-ключ OpenAI (или совместимого провайдера)

## Установка

```bash
# Перейти в директорию
cd tg-bot-chain

# Создать виртуальное окружение
python -m venv .venv

# Активировать (Windows PowerShell)
.\.venv\Scripts\Activate.ps1

# Активировать (Linux / macOS)
source .venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

## Настройка

Создайте файл `.env` по образцу `.env.example`:

```env
OPENAI_API_KEY=sk-...
MODEL_NAME=gpt-5.4-mini-2026-03-17
BOT_TOKEN=123456:ABC-DEF...
```

| Переменная | Обязательна | По умолчанию | Описание |
|-----------|-------------|--------------|----------|
| `OPENAI_API_KEY` | ✅ | — | Ключ API OpenAI |
| `MODEL_NAME` | ❌ | `gpt-5.4-mini-2026-03-17` | Модель для генерации |
| `BOT_TOKEN` | ❌* | — | Токен бота от @BotFather |

*\* Не нужен для генерации, но нужен для запуска сгенерированного бота.*

## Использование

```bash
# Базовое использование
python script.py "Описание вашего бота"

# С подробным выводом
python script.py -v "Описание вашего бота"
```

### Примеры

```bash
# Бот с мемами и кнопками
python script.py "Бот, который отправляет случайные мемы. Команды дублируются кнопками"

# Бот-напоминалка
python script.py "Бот для создания напоминаний с указанием времени"

# Бот-переводчик
python script.py "Бот, который переводит текст с русского на английский и обратно"

# Бот-опросник
python script.py "Бот для проведения опросов с инлайн-кнопками и подсчётом голосов"
```

### Флаг `--verbose` / `-v`

В verbose-режиме отображаются:
- Результат анализа (команды, обработчики, зависимости)
- Фрагмент сгенерированного кода
- Детали ревью и исправлений
- Время каждого этапа

Полный лог всегда сохраняется в `generation.log`.

## Результат

После выполнения скрипт создаёт файл `generated_bot.py` в текущей директории.

```bash
# Запуск сгенерированного бота
python generated_bot.py
```

Убедитесь, что `BOT_TOKEN` установлен в `.env`.

## Архитектура цепочек

| Цепочка | Temperature | Назначение |
|---------|-------------|------------|
| `analysis_chain` | 0.1 | Точный анализ задания |
| `code_chain` | 0.2 | Генерация кода с вариативностью |
| `review_chain` | 0.0 | Детерминированная проверка |
| `fix_chain` | 0.0 | Целенаправленное исправление ошибок |

### Почему разная температура?

- **0.0** — максимально предсказуемый ответ. Для ревью и фиксов важна точность, а не креативность.
- **0.1** — почти детерминированный, но с минимальной гибкостью формулировок.
- **0.2** — небольшая вариативность для кода: разные стили реализации одной логики.

## Стек технологий

- **LangChain** — оркестрация LLM-цепочек
- **OpenAI API** — генерация и анализ кода
- **aiogram 3.7+** — целевой фреймворк генерируемых ботов
- **Python ast** — валидация синтаксиса и проверка импортов
- **python-dotenv** — управление переменными окружения

## Совместимость с другими LLM

Скрипт использует `ChatOpenAI`, который совместим с любым OpenAI-совместимым API:

```bash
set OPENAI_API_BASE=http://localhost:1234/v1   # например, LM Studio
set MODEL_NAME=llama-3
```
